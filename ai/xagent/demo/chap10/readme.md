### 最小可用的记忆系统长什么样
如果你现在要给自己的 Agent 加记忆能力，不需要一步到位做成 Mem0 那种完整 pipeline。一个最小可用版本：
- 第一步：给 Agent 一个写记忆的工具。 定义一个 save_memory 工具，让 Agent 在对话中遇到值得长期保存的信息时主动调用。存储介质可以是文件，也可以是数据库。
- 第二步：每轮对话开始时注入记忆索引。 把已有记忆的标题和摘要列表塞进 system prompt。Agent 看到索引后，如果觉得某条记忆跟当前问题相关，就主动去读全文。Claude Code 的"MEMORY.md 索引 + 按需读取正文"就是这个模式。
- 第三步：定期做清理和合并。 重复的条目合并，过时的条目标记过期。可以手动做，也可以写个脚本定期跑。
这三步加起来，已经能让 Agent 从"每次失忆"变成"基本能记住重要的事"。向量检索、图谱推理、自动提取这些，等跑起来之后遇到具体瓶颈再加不迟。

很多人一开始会把“历史消息”和“长期记忆”混在一起。真正做过一段时间之后就会发现，这两者的职责完全不同。
- 会话历史，保存的是一次对话过程
- 项目记忆，沉淀的是跨对话仍然成立的关键信息

记忆文件要带 frontmatter

```
---
name: Testing policy
description: integration tests must run against a real database
type: feedback
---
```
